SeaTraffic plugin for X-Plane®

Overview

This X-Plane plugin displays ships moving along real-life routes.

Requirements

This plugin can be used with X-Plane 9 or later. It relies on the OpenSceneryX Object Library for its ship models.

Installation

  • If you do not already have the OpenSceneryX Object Library installed, download and install it.
  • Copy or move the SeaTraffic folder from within this .zip archive into the X-Plane/Resources/plugins folder.

Adding / modifying routes

This plugin uses ferry routes extracted from the OpenStreetMap project. If you find that a ferry route is missing or incorrect you can sign up to the project and add or modify the route. Start by reading the Beginners' guide and other info on the Main Page.

The OpenStreetMap "way" tags for creating a ferry route are described here and here. This plugin decides which type of ship to send along a ferry route based on the following tags:

  • route = ferry
  • name = route name
  • ref = line number or route code (if applicable)
  • foot = yes / no
  • motorcar = yes / no
  • hgv = yes / no

This plugin recognises tourist boat routes and cruise ship lines by the following additional tag:

  • ferry = tourist / cruise

Getting your new / modified route into X-Plane

This plugin doesn't read ferry routes directly from the OpenStreetMap servers (that would be too slow). Instead, it contains a pre-computed database of routes. In order to visualise your new or modified ferry route(s) in X-Plane you can either:

  • wait for the author to release a new version of the plugin, or
  • wait about an hour for your edits to appear on the OpenStreetMap servers, then re-generate the plugin's database by running the script X-Plane/Resources/plugins/SeaTraffic/buildroutes.py (Windows users must install Python 2.x in order to run this script).
